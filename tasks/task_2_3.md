# Задача 2.3: Раздел Services и структурный раздел Projects (без реальных проектов)

## Связь с юзер-кейсами
- UC-05: Просмотр проектов и переход на GitHub / внешние сайты (структура раздела).
- Дополнительный бизнес-блок «Услуги / Чем могу быть полезен» (расширение UC-03/UC-05).

## Цель задачи
Добавить:
- раздел «Услуги» с перечислением типов задач, которые вы выполняете,
- раздел «Проекты» с готовой структурой карточек и модального окна,
но без обязательного наполнения реальными проектами (можно использовать 1–2 примера/заглушки или оставить пустые состояния).

## Описание изменений

### Новые файлы
- `src/entities/services/model/services.ts` — список `ServiceItem[]`:
  - каждая услуга: название + описание (LocalizedText);
  - примеры: разработка SPA на Vue/React, миграция фронтенда, аудит и оптимизация производительности и т.п.

- `src/entities/projects/model/projects.ts` — список `ProjectItem[]` (может быть пустым на первом этапе, но структура должна быть описана):
  - `title: LocalizedText`
  - `shortDescription: LocalizedText`
  - `techStack: string[]`
  - `githubUrl?: string`
  - `liveUrl?: string`
  - `details?: LocalizedText`

### Изменения в существующих файлах

- `src/widgets/services/ui/ServicesSection.vue`
  - Подключить `services.ts`;
  - Отрисовать список услуг (карточки/плитки с названием и описанием);
  - Использовать i18n для заголовка раздела.

- `src/widgets/projects/ui/ProjectsSection.vue`
  - Подключить `projects.ts`;
  - Если список пуст:
    - отобразить "пустое состояние" с текстом вроде «Проекты будут добавлены позже» (RU/EN);
  - Если в будущем список не пуст:
    - отрисовать карточки проектов с кнопками «Подробнее», «GitHub», «Сайт»;
    - использовать модальное окно для отображения детального описания.

## Тест-кейсы

### End-to-end тесты
1. **TC-E2E-15:** Отображение раздела «Услуги»
   - Входные данные: открыть раздел «Услуги».
   - Ожидаемый результат:
     - отображается список из нескольких услуг;
     - каждая услуга имеет заголовок и краткое описание;
     - на мобильных карточки читаемы и не «ломаются».

2. **TC-E2E-16:** Отображение раздела «Проекты» при пустом списке
   - Входные данные: при пустом `projects.ts` открыть раздел «Проекты».
   - Ожидаемый результат:
     - отображается понятное «пустое состояние» с текстом о том, что проекты будут добавлены позже;
     - UX не выглядит «сломленным» (нет пустого блока без текста).

3. **TC-E2E-17:** Переключение языка для разделов Services/Projects
   - Входные данные:
     - переключить язык на EN;
     - открыть разделы «Услуги» и «Проекты».
   - Ожидаемый результат:
     - заголовки и описания услуг на английском;
     - текст «пустого состояния» в Projects также локализован.

## Критерии приёмки
- [ ] Раздел «Услуги» отображает понятный набор услуг с RU/EN текстами.
- [ ] Раздел «Проекты» отображает корректное «пустое состояние» при отсутствии проектов.
- [ ] При переключении языка тексты обоих разделов корректно меняются.

## Примечания
- Реальные проекты и их карточки будут добавлены позже, без изменения архитектуры. Настоящие данные можно будет подставить в `projects.ts`.
