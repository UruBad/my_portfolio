# Задача 2.2: Разделы Experience, Skills, Education (RU/EN)

## Связь с юзер-кейсами
- UC-04: Просмотр опыта работы (timeline).
- UC-06: Просмотр навыков и стек технологий.
- UC-07: Просмотр образования и курсов.

## Цель задачи
Заполнить разделы «Опыт», «Навыки» и «Образование и курсы» данными на основе `tasks/resume.pdf` и ТЗ, используя модель данных из архитектуры (ExperienceItem, SkillCategory, EducationItem, CourseItem) с поддержкой двух локалей (RU/EN).

## Описание изменений

### Новые файлы
- `src/entities/experience/model/experience.ts` — список `ExperienceItem[]`:
  - включает все ключевые места работы из резюме с периодами, позициями и краткими описаниями.
- `src/entities/skills/model/skills.ts` — список `SkillCategory[]`:
  - категории: «Языки и платформы», «Фреймворки и библиотеки», «Инструменты и инфраструктура», «Верстка» и т.п.;
  - наполнение на основе раздела навыков из резюме.
- `src/entities/education/model/education.ts` — списки `EducationItem[]` и `CourseItem[]`:
  - высшее образование (СКГМИ);
  - курсы/сертификаты (React, Next.js, Nest.js и т.д.).

### Изменения в существующих файлах

- `src/widgets/experience/ui/ExperienceSection.vue`
  - Подключить `experience.ts`;
  - Отрисовать опыт в виде списка карточек или таймлайна:
    - сортировка по дате (новые сверху);
    - название компании, позиция, период, краткое описание;
    - при желании — список проектов/технологий.

- `src/widgets/skills/ui/SkillsSection.vue`
  - Подключить `skills.ts`;
  - Отобразить категории и входящие навыки (бейджи/теги);
  - Для статических заголовков использовать i18n.

- `src/widgets/education/ui/EducationSection.vue`
  - Подключить `education.ts`;
  - Отрисовать:
    - блок «Высшее образование»;
    - блок «Курсы и сертификаты».

### Интеграция локализации

- Для описаний опыта (`description`), названий категорий и курсов использовать `LocalizedText` (RU/EN).
- Заголовки разделов и подписи (например, «Технологии», «Период», «Город») берутся из i18n‑словаря.

## Тест-кейсы

### End-to-end тесты
1. **TC-E2E-11:** Отображение опыта работы
   - Входные данные: открыть раздел «Опыт».
   - Ожидаемый результат:
     - все ключевые места работы из резюме присутствуют;
     - порядок — от последнего опыта к первому;
     - для каждой записи видны компания, позиция и период.

2. **TC-E2E-12:** Отображение навыков
   - Входные данные: открыть раздел «Навыки».
   - Ожидаемый результат:
     - навыки из резюме распределены по категориям;
     - нет критичных дубликатов (JavaScript / JS можно объединять логически);
     - верстка адаптивна.

3. **TC-E2E-13:** Отображение образования и курсов
   - Входные данные: открыть раздел «Образование и курсы».
   - Ожидаемый результат:
     - отображаются данные о высшем образовании;
     - отображаются ключевые курсы/сертификаты 2023 года;
     - при наличии ссылок на сертификаты они кликабельны.

4. **TC-E2E-14:** Переключение языка для Experience/Skills/Education
   - Входные данные:
     - переключить язык на EN;
     - открыть соответствующие разделы.
   - Ожидаемый результат:
     - заголовки и подписи на английском;
     - описания опыта/курсов (если локализованы) на английском.

### Модульные тесты
- Необязательны; основная проверка — визуальная и логическая (покрытие юзер-кейсов).

### Регрессионные тесты
- Проверка, что разделы Hero/About/Contacts (задача 2.1) работают как раньше и не ломаются после добавления новых данных.

## Критерии приёмки
- [ ] Весь опыт из резюме представлен и отсортирован по времени.
- [ ] Навыки сгруппированы по понятным категориям.
- [ ] Образование и курсы корректно отображаются.
- [ ] Переключение языка отражается в заголовках и основных текстах этих разделов.

## Примечания
- При необходимости данные могут быть расширены вручную без изменения архитектуры (добавлением новых элементов в массивы).
