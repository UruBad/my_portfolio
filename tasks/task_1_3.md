# Задача 1.3: Реализация темизации (light/dark) и базовых анимаций появления секций

## Связь с юзер-кейсами
- UC-01: Главный экран (визуальный стиль).
- UC-02: Навигация по разделам (анимации при появлении секций).
- UC-09: Переключение темы (светлая/тёмная) и базовые анимации.

## Цель задачи
Добавить механизм управления темой (light/dark), реализовать переключатель темы в хедере с сохранением выбора в `localStorage`, а также добавить базовые CSS-анимации появления секций при прокрутке.

## Описание изменений

### Новые файлы
- `src/shared/config/theme/palette.scss` — определение CSS-переменных для светлой и тёмной темы (цвет фона, текста, акцентный цвет и т.д.).
- `src/shared/lib/theme/useTheme.ts` — composable для работы с темой:
  - хранит текущую тему;
  - предоставляет функции `setTheme(theme)`, `toggleTheme()`, `initTheme()`.
- `src/features/theme-toggle/ui/ThemeToggle.vue` — компонент переключателя темы (иконка солнца/луны).
- `src/shared/lib/viewport/useInViewAnimation.ts` — composable для отслеживания появления секции в viewport (на базе `IntersectionObserver`).

### Изменения в существующих файлах

- `src/app/styles/index.scss`
  - Подключение палитры `palette.scss`.
  - Установка базовых стилей для `body` с использованием CSS-переменных (`--bg-color`, `--text-color`, `--accent-color` и т.д.).

- `src/app/App.vue`
  - Инициализация темы при монтировании приложения (`initTheme()`).

- `src/widgets/header/ui/Header.vue`
  - Добавление компонента `ThemeToggle` в правую часть хедера рядом с переключателем языка.

- `src/widgets/*Section.vue`
  - Добавление использования `useInViewAnimation`:
    - связывание `ref` корневого элемента секции с composable;
    - применение CSS-класса, который запускает анимацию при появлении секции.

### Интеграция компонентов

- `useTheme`:
  - читает тему из `localStorage` (`portfolio_theme`) или из системных настроек (`prefers-color-scheme`) при первом запуске;
  - применяет CSS-класс/атрибут на `document.documentElement` (`data-theme="light"` / `data-theme="dark"`), к которому привязаны CSS-переменные.

- `ThemeToggle`:
  - отображает текущее состояние (иконка солнца для light, луны для dark);
  - по клику вызывает `toggleTheme()`.

- `useInViewAnimation`:
  - использует `IntersectionObserver` для установки флага `isInView`;
  - возвращает объект с `isInView` и классом, который можно навесить на корневой блок секции (`section--visible`).

## Тест-кейсы

### End-to-end тесты
1. **TC-E2E-04:** Переключение темы и сохранение выбора
   - Входные данные:
     - открыть приложение в браузере;
     - нажать на переключатель темы.
   - Ожидаемый результат:
     - основной фон и цвет текста меняются в соответствии с новой темой;
     - после перезагрузки страницы сохраняется выбранная тема.

2. **TC-E2E-05:** Инициализация темы из системных настроек
   - Входные данные:
     - очистить `localStorage` для ключа `portfolio_theme`;
     - в системных настройках OS включена тёмная тема;
     - открыть приложение.
   - Ожидаемый результат:
     - по умолчанию выбирается тёмная тема (если такая логика реализована в `initTheme()` и зафиксирована в документации).

3. **TC-E2E-06:** Анимации появления секций
   - Входные данные:
     - открыть приложение;
     - прокрутить страницу сверху вниз.
   - Ожидаемый результат:
     - при первом появлении каждой секции в видимой области она плавно «проявляется» (fade/slide);
     - при повторной прокрутке анимация не мешает чтению (можно либо не перезапускать, либо сделать её менее заметной).

### Модульные тесты
- На этом этапе достаточно ручной проверки в браузере; при необходимости можно позже добавить unit‑тест на корректную инициализацию темы (`useTheme`).

### Регрессионные тесты
- Убедиться, что после добавления темизации и анимаций:
  - приложение по-прежнему стартует без ошибок;
  - структура секций и локализация (Задачи 1.1 и 1.2) не сломаны.

## Критерии приёмки
- [ ] Переключатель темы отображается в хедере и меняет тему по клику.
- [ ] Выбранная тема сохраняется и восстанавливается при перезагрузке страницы.
- [ ] При скролле секции появляются с аккуратной анимацией.
- [ ] Анимации не вызывают визуальных артефактов и миганий.

## Примечания
- Детальный подбор цветовой палитры (оттенки, контраст) можно улучшить на этапе задачи 3.1 (полировка дизайна и доступности); сейчас важно заложить механизм и базовый стиль.
