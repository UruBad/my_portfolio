# Задача 1.2: Реализация i18n (ru/en) и переключателя языка (заглушечные тексты)

## Связь с юзер-кейсами
- Все UC: базовая локализация интерфейса (RU/EN) для навигации и статических текстов.

## Цель задачи
Добавить в проект механизм локализации с поддержкой двух языков (`ru`, `en`), реализовать переключатель языка в хедере и обеспечить хранение выбранной локали в `localStorage`. На этом этапе тексты могут быть частично заглушечными, но структура i18n должна быть готова для дальнейшего наполнения.

## Описание изменений

### Новые файлы
- `src/shared/config/i18n/locales.ts` — типы локали, ключи и вспомогательные типы.
- `src/shared/config/i18n/dictionary.ts` — словари переводов для `ru` и `en` (основные ключи навигации, простые подписи и т.д.).
- `src/shared/lib/i18n/useI18n.ts` — composable для работы с локализацией:
  - хранит текущую локаль;
  - предоставляет функции `setLocale(locale)` и `t(key)`.
- `src/features/language-switcher/ui/LanguageSwitcher.vue` — компонент переключателя языка (кнопки/тоггл ru/en).

### Изменения в существующих файлах

- `src/app/App.vue`
  - Оборачивает всё приложение в провайдера i18n (через `provide`/`inject` или использование composable на верхнем уровне).

- `src/widgets/header/ui/Header.vue`
  - Заменяет захардкоженные строки навигации на вызовы `t('nav.about')`, `t('nav.experience')` и т.п.
  - Добавляет компонент `LanguageSwitcher` в правую часть хедера.

- `src/widgets/...Section.vue`
  - Для заголовков секций добавляет использование `t('section.about.title')`, `t('section.experience.title')` и т.д.
  - Тексты внутри секций пока могут быть плейсхолдерами, но подключёнными к i18n (например, `t('section.about.placeholder')`).

### Интеграция компонентов

- `useI18n`:
  - при инициализации пытается прочитать локаль из `localStorage` (`portfolio_locale`) и, если нет значения, использует `navigator.language` (ru/en) с дефолтом `ru`;
  - при смене языка записывает значение в `localStorage` и триггерит реактивное обновление.

- `LanguageSwitcher`:
  - отображает две кнопки/тоггла (RU / EN);
  - подсвечивает активный язык;
  - при клике вызывает `setLocale('ru')` или `setLocale('en')`.

## Тест-кейсы

### End-to-end тесты
1. **TC-E2E-02:** Переключение языка интерфейса
   - Входные данные:
     - открыть приложение в браузере;
     - по умолчанию язык `ru`;
     - нажать на `EN`.
   - Ожидаемый результат:
     - текст пунктов меню и заголовков секций меняется на английский;
     - при перезагрузке страницы язык остаётся `en`.

2. **TC-E2E-03:** Выбор языка по умолчанию
   - Входные данные:
     - очистить `localStorage` для ключа `portfolio_locale`;
     - открыть приложение в браузере.
   - Ожидаемый результат:
     - если язык системы `ru` — интерфейс стартует в `ru`;
     - иначе — интерфейс стартует в `en` (или `ru`, в зависимости от выбранной логики; зафиксировать в реализации и документации).

### Модульные тесты
- Опционально: можно добавить простой unit‑тест для `useI18n`, проверяющий:
  - установку локали;
  - возврат корректного перевода по ключу.

### Регрессионные тесты
- Проверить, что структура секций и базовый layout из Задачи 1.1 не сломались:
  - страница всё так же загружается без ошибок;
  - все секции продолжают отображаться.

## Критерии приёмки
- [ ] Переключатель языка отображается в хедере.
- [ ] При смене языка обновляются тексты навигации и заголовков секций.
- [ ] Выбранная локаль сохраняется и восстанавливается при перезагрузке страницы.
- [ ] Не появляются ошибки в консоли при переключении языков.

## Примечания
- На этом этапе можно перевести только ключевые элементы UI (навигация, заголовки секций, базовые подписи). Наполнение контентом (резюме, опыт, услуги) будет выполнено в задачах Этапа 2.
