# Задача 3.1: Полировка анимаций, адаптивности и доступности

## Связь с юзер-кейсами
- UC-01–UC-10: все (нефункциональные требования: дизайн, адаптивность, доступность, анимации).

## Цель задачи
Довести интерфейс до состояния, соответствующего нефункциональным требованиям из ТЗ:
- современный, аккуратный дизайн,
- корректная адаптивная верстка для основных брейкпоинтов,
- базовая доступность (контраст, фокус, aria‑атрибуты),
- согласованные анимации.

## Описание изменений

### Новые файлы
- При необходимости можно добавить:
  - `src/shared/styles/breakpoints.scss` — общие константы брейкпоинтов.
  - `src/shared/styles/typography.scss` — базовые настройки типографики.

### Изменения в существующих файлах

- `src/app/styles/index.scss`, `palette.scss`
  - Уточнение цветовой палитры для повышения контраста (особенно в тёмной теме).
  - Настройка размеров шрифтов и отступов для разных брейкпоинтов.

- Секции (`HeroSection`, `AboutSection`, `ExperienceSection`, `SkillsSection`, `ProjectsSection`, `EducationSection`, `ServicesSection`, `ContactsSection`)
  - Проверка и корректировка верстки на:
    - десктоп (≥1280px),
    - ноутбук (1024–1280px),
    - планшет (768–1024px),
    - мобильные (≤768px).
  - Устранение потенциальных горизонтальных скроллов.
  - Обеспечение адекватной высоты строк и отступов для удобства чтения.

- Анимации
  - Убедиться, что классы анимации применяются единообразно (без дерганий).
  - При необходимости ограничить анимации для пользователей с `prefers-reduced-motion`.

- Доступность
  - Проверить, что все интерактивные элементы доступны по Tab.
  - Добавить `aria-label` для иконок (переключатель языка, темы, иконки GitHub и т.п.).
  - Убедиться, что контраст текста и фона соответствует базовым рекомендациям (можно использовать встроенные средства браузера/плагины).

## Тест-кейсы

### End-to-end тесты (визуальная/ручная проверка)
1. **TC-E2E-18:** Проверка адаптивности
   - Входные данные:
     - открыть сайт на разных ширинах (320, 768, 1024, 1440 px);
   - Ожидаемый результат:
     - верстка не «ломается»;
     - элементы не уползают за границы экрана;
     - текст читаем.

2. **TC-E2E-19:** Проверка доступности фокусировки
   - Входные данные:
     - использовать клавишу Tab для навигации.
   - Ожидаемый результат:
     - фокус проходит по всем важным интерактивным элементам;
     - виден фокус‑стейт (outline/подсветка).

3. **TC-E2E-20:** Проверка анимаций
   - Входные данные:
     - прокрутить страницу несколько раз вверх/вниз;
   - Ожидаемый результат:
     - анимации не избыточны, не мешают чтению;
     - при включенном `prefers-reduced-motion` (если обрабатывается) анимации минимальны или отключены.

## Критерии приёмки
- [ ] Сайт корректно отображается на основных брейкпоинтах.
- [ ] Нет горизонтального скролла в стандартных сценариях.
- [ ] Весь текст хорошо читаем, контраст достаточен.
- [ ] Все основные интерактивные элементы доступны по клавиатуре и имеют видимый фокус.
- [ ] Анимации аккуратны и не мешают использованию сайта.
