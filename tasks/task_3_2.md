# Задача 3.2: Настройка GitHub Pages и деплой

## Связь с юзер-кейсами
- Все UC: доступность сайта по публичному URL.

## Цель задачи
Настроить сборку и деплой проекта на GitHub Pages, чтобы сайт портфолио был доступен по адресу вида `https://<username>.github.io/portfolio/`.

## Описание изменений

### Новые/изменённые файлы

- `vite.config.ts`
  - Установить `base: '/portfolio/'`, чтобы корректно работали пути к ассетам на GitHub Pages.

- `package.json`
  - Добавить скрипты:
    - `"build": "vite build"`
    - `"preview": "vite preview"`
    - опционально: `"deploy": "npm run build && <скрипт деплоя на gh-pages>"`

- GitHub Actions workflow (опционально, если хочешь автоматизировать деплой)
  - `/.github/workflows/deploy.yml` с шагами:
    - `checkout`, `setup-node`, `npm ci`, `npm run build`, деплой на GitHub Pages (используя `actions/deploy-pages` или `peaceiris/actions-gh-pages`).

### Инструкции по ручному деплою

Описать в `README.md` (или отдельном разделе) шаги:
1. Убедиться, что репозиторий `portfolio` находится на GitHub.
2. Выполнить:
   - `npm install`
   - `npm run build`
3. Задеплоить содержимое `dist` в ветку `gh-pages` (вручную или через actions).
4. В настройках репозитория включить GitHub Pages из ветки `gh-pages` (root).

## Тест-кейсы

### End-to-end тесты (после деплоя)
1. **TC-E2E-21:** Доступность сайта по URL GitHub Pages
   - Входные данные:
     - открыть `https://<username>.github.io/portfolio/` в браузере.
   - Ожидаемый результат:
     - сайт успешно загружается;
     - работают все разделы и навигация;
     - ссылки на резюме/контакты/GitHub функционируют.

2. **TC-E2E-22:** Проверка работы относительных путей
   - Входные данные:
     - обновить страницу на внутренних якорях (например, `#projects`);
   - Ожидаемый результат:
     - сайт корректно перезагружается и отображает нужный раздел;
     - ресурсы (CSS/JS/изображения) подгружаются без 404.

## Критерии приёмки
- [ ] Сайт успешно собирается командой `npm run build`.
- [ ] Сборка разворачивается на GitHub Pages и доступна по публичному URL.
- [ ] При открытии/обновлении страницы ресурсы подтягиваются корректно (нет 404 по ассетам).

## Примечания
- Конкретный способ деплоя (скрипт, GitHub Actions, ручной) можно выбрать на своё усмотрение; важно, чтобы он был описан и воспроизводим.
