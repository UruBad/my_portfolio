# Задача 1.1: Инициализация Vue 3 + TS проекта и базовой FSD-структуры

## Связь с юзер-кейсами
- UC-01: Просмотр главного экрана портфолио (каркас страницы и базовая разметка).
- UC-02: Навигация по разделам (базовая структура секций).

## Цель задачи
Создать новый проект на Vue 3 + TypeScript + Vite, настроить базовую структуру каталогов в стиле FSD, добавить заготовки основных секций страницы (без реального контента и логики), чтобы получить end-to-end отображение страницы с пустыми блоками.

## Описание изменений

### Новые файлы
- `package.json` — конфигурация npm-проекта.
- `vite.config.ts` — конфигурация Vite.
- `tsconfig.json`, `tsconfig.app.json` — настройки TypeScript.
- `index.html` — корневой HTML-файл.
- Структура `src/`:
  - `src/main.ts` — точка входа приложения.
  - `src/app/App.vue` — корневой компонент приложения.
  - `src/app/providers/index.ts` — провайдеры приложения (тема, i18n будут добавлены далее).
  - `src/app/styles/index.scss` — глобальные стили и базовые переменные.
  - `src/pages/portfolio/PagePortfolio.vue` — основная страница портфолио.
  - `src/widgets/header/ui/Header.vue` — заготовка хедера с навигацией по секциям (пока без логики скролла).
  - `src/widgets/footer/ui/Footer.vue` — заготовка футера.
  - `src/widgets/hero/ui/HeroSection.vue` — заготовка главного экрана.
  - `src/widgets/about/ui/AboutSection.vue` — заготовка секции «Обо мне».
  - `src/widgets/experience/ui/ExperienceSection.vue` — заготовка секции «Опыт».
  - `src/widgets/skills/ui/SkillsSection.vue` — заготовка секции «Навыки».
  - `src/widgets/projects/ui/ProjectsSection.vue` — заготовка секции «Проекты».
  - `src/widgets/education/ui/EducationSection.vue` — заготовка секции «Образование и курсы».
  - `src/widgets/services/ui/ServicesSection.vue` — заготовка секции «Услуги».
  - `src/widgets/contacts/ui/ContactsSection.vue` — заготовка секции «Контакты».
  - `src/shared/ui/Section/Section.vue` — базовый обёрточный компонент для секций (общие отступы и ширина).

### Изменения в существующих файлах
- Отсутствуют (проект создаётся с нуля, кроме уже имеющихся служебных файлов `agents/`, `tasks/`, `status.md`, `command.md`, которые не изменяются).

### Интеграция компонентов
- `main.ts` монтирует `App.vue` в элемент `#app`.
- `App.vue`:
  - подключает глобальные стили;
  - рендерит `Header`, `PagePortfolio`, `Footer`.
- `PagePortfolio.vue` рендерит секции в следующем порядке:
  - `HeroSection`
  - `AboutSection`
  - `ExperienceSection`
  - `SkillsSection`
  - `ProjectsSection`
  - `EducationSection`
  - `ServicesSection`
  - `ContactsSection`
- Каждая секция получает уникальный `id` (`hero`, `about`, `experience`, `skills`, `projects`, `education`, `services`, `contacts`) для дальнейшей навигации.

## Тест-кейсы

### End-to-end тесты
1. **TC-E2E-01:** Старт приложения и отображение пустых секций
   - Входные данные: запуск `npm install`, затем `npm run dev`, открытие приложения в браузере.
   - Ожидаемый результат:
     - страница загружается без ошибок JS в консоли;
     - видны заглушки всех секций (заголовки или простые блоки с текстом «Section placeholder»);
     - нет горизонтального скролла на десктопе при базовом размере окна.
   - Примечание: на этом этапе контент и локализация не реализованы, проверяется только каркас.

### Модульные тесты
- На этом этапе модульные тесты не обязательны; достаточно ручной проверки в браузере, т.к. задача создаёт только каркас. Тестовая инфраструктура (если потребуется) будет добавлена на последующих задачах.

### Регрессионные тесты
- Так как проект создаётся с нуля, регрессионные тесты отсутствуют. Достаточно убедиться, что приложение стартует без ошибок.

## Критерии приёмки
- [ ] Успешно выполнена установка зависимостей (`npm install`) без критичных ошибок.
- [ ] Команда `npm run dev` поднимает дев-сервер без ошибок.
- [ ] В браузере отображаются все секции в виде заглушек.
- [ ] Структура каталогов `src/app`, `src/pages`, `src/widgets`, `src/shared` создана и соответствует описанию.
- [ ] Не изменены и не удалены существующие служебные файлы в корне репозитория (`agents/`, `tasks/`, `status.md`, `command.md`).

## Примечания
- На этом этапе **не реализуются** i18n, темизация, реальные данные из резюме, анимации и навигация по клику — только структура и базовый layout.
