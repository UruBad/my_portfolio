## Важные замечания (🟡 MAJOR)

1. **Нет явного описания стратегии для будущего расширения раздела «Проекты»**

**Местоположение:** Раздел 4. Модель данных / сущность `ExperienceProject` и раздел 2.1 `ProjectsSection`.  
**Проблема:** В ТЗ указано, что проекты будут добавлены позже, сейчас раздел запускается в виде плейсхолдера. В архитектуре не описано явно, как будут объединяться данные из резюме и GitHub (например, по списку репозиториев или вручную через конфигурацию).  
**Рекомендация:**  
Добавить в модель данных или раздел «Открытые вопросы» явное описание стратегии:
- либо полностью конфигурируемый список проектов в коде;
- либо полуавтоматическая загрузка списка из GitHub API (на последующих этапах, с указанием ограничений).

## Незначительные замечания (🟢 MINOR)

1. **Не описана структура словарей i18n по уровням FSD**

**Местоположение:** Раздел 5. Интерфейсы / локализация.  
**Рекомендация:**  
Добавить краткое соглашение, что:
- глобальные фразы (nav, footer, общие кнопки) — в `shared/config/i18n`;
- доменные тексты (описания опыта, услуг и т.п.) — внутри соответствующих `entities/*/model`, используя `LocalizedText`.
