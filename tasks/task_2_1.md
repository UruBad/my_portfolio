# Задача 2.1: Разделы Hero, About, Contacts и скачивание резюме (RU/EN)

## Связь с юзер-кейсами
- UC-01: Просмотр главного экрана портфолио.
- UC-03: Просмотр раздела «Обо мне» и ключевой информации.
- UC-08: Просмотр контактов и быстрые действия.
- UC-10: Скачивание резюме в формате PDF.

## Цель задачи
Заполнить данными из `tasks/resume.pdf` и ТЗ разделы:
- Hero (главный экран),
- About («Обо мне»),
- Contacts (контакты),
и реализовать функциональность скачивания резюме (PDF) через CTA‑кнопки, с поддержкой RU/EN локалей.

## Описание изменений

### Новые файлы
- `public/resume.pdf` — копия актуального резюме (для скачивания с фронтенда).
- `src/entities/person/model/profile.ts` — данные профиля (`PersonProfile`) с полями:
  - `fullName`,
  - `position`,
  - `location`,
  - `salaryExpectationUSD`,
  - `summary: LocalizedText`.
- `src/entities/contacts/model/contacts.ts` — данные контактов (`ContactInfo`):
  - `phone`,
  - `email`,
  - `githubUrl`,
  - `location`.
- `src/features/download-resume/lib/useDownloadResume.ts` — helper/composable для скачивания резюме:
  - открывает `/resume.pdf` в новой вкладке или инициирует скачивание.
- `src/features/download-resume/ui/DownloadResumeButton.vue` — кнопка «Скачать резюме».

### Изменения в существующих файлах

- `src/widgets/hero/ui/HeroSection.vue`
  - Подключить данные `PersonProfile` и `ContactInfo`.
  - Отрисовать:
    - имя и позицию;
    - короткое summary (используя текущую локаль через `t` или напрямую из `LocalizedText`);
    - CTA‑кнопки:
      - «Скачать резюме» (использует `DownloadResumeButton`);
      - «Написать» (mailto: с e‑mail из контактов);
      - «GitHub» (ссылка на профиль).

- `src/widgets/about/ui/AboutSection.vue`
  - Использовать `PersonProfile.summary` для отображения текста «Обо мне» в выбранной локали.
  - Добавить вывод:
    - города,
    - информации о готовности к переезду/командировкам,
    - целевой позиции и уровня дохода (сформулировать на основе резюме).

- `src/widgets/contacts/ui/ContactsSection.vue`
  - Отобразить:
    - номер телефона (tel:),
    - e‑mail (mailto:),
    - GitHub,
    - город.
  - Подписывать поля с учётом локали (RU/EN).

### Интеграция компонентов

- Для текстов, которые являются статическими фразами (подписи «Телефон», «Email», «GitHub», подписи кнопок и т.п.), использовать i18n‑словарь.
- Для контентных текстов из резюме использовать структуру `LocalizedText` в сущностях:
  - на первом этапе можно заполнить `ru` полноценным текстом и `en` чуть более лаконичной версией, основанной на резюме.

## Тест-кейсы

### End-to-end тесты
1. **TC-E2E-07:** Отображение Hero‑блока
   - Входные данные:
     - открыть приложение с локалью `ru`.
   - Ожидаемый результат:
     - отображаются имя и позиция;
     - виден краткий текст «Обо мне»;
     - видны три CTA‑кнопки (скачать резюме, написать, GitHub).

2. **TC-E2E-08:** Скачивание резюме
   - Входные данные:
     - нажать на кнопку «Скачать резюме».
   - Ожидаемый результат:
     - браузер открывает или скачивает файл `resume.pdf`;
     - в консоли нет ошибок 404 по этому пути.

3. **TC-E2E-09:** Отображение контактов и работа ссылок
   - Входные данные:
     - перейти к разделу «Контакты».
   - Ожидаемый результат:
     - отображаются телефон, e‑mail, GitHub, город;
     - клик по телефону пытается открыть звонилку (tel:);
     - клик по e‑mail — mailto:;
     - клик по GitHub — открывает профиль в новой вкладке.

4. **TC-E2E-10:** Переключение языка для Hero/About/Contacts
   - Входные данные:
     - открыть приложение;
     - переключить язык на EN;
     - прокрутить к секциям Hero, About, Contacts.
   - Ожидаемый результат:
     - заголовки и статические подписи на английском;
     - тексты summary и «Обо мне» отображаются на английском из `LocalizedText`.

### Модульные тесты
- Опционально: тест на `useDownloadResume`, проверяющий корректное формирование URL.

### Регрессионные тесты
- Убедиться, что механизмы i18n и темизации из задач 1.2 и 1.3 не сломаны.

## Критерии приёмки
- [ ] В Hero‑блоке корректно отображаются данные профиля.
- [ ] Работают CTA‑кнопки «Скачать резюме», «Написать», «GitHub».
- [ ] Раздел «Обо мне» заполняется данными из резюме (RU/EN).
- [ ] Раздел «Контакты» отображает все ключевые контакты и ссылки.
- [ ] При переключении языка все три секции отображают корректный язык.
